# 客户数据脱敏实施方案

## 背景说明

根据《个人信息保护法》和银保监会相关要求，需要对客户敏感数据进行脱敏处理，确保在知识库检索和展示环节不泄露个人隐私信息。

## 脱敏规则

### 手机号脱敏

规则：保留前 3 位和后 4 位，中间用 * 替代
示例：13812345678 → 138****5678

### 身份证号脱敏

规则：整段替换为脱敏标识
示例：110101199001011234 → 身份证号：已脱敏

### 金额脱敏

规则：按区间替换
示例：250万元 → 金额区间：100万-300万

### 客户名称脱敏

规则：通过预定义的客户名称词典匹配替换
示例：平安银行 → 某金融客户

## 技术实现

### 正则引擎

采用 Java 正则表达式实现实时脱敏，支持：

- 手机号：`1[3-9]\\d{9}`
- 邮箱：`\\w+@\\w+\\.\\w+`
- 身份证：`\\d{17}[\\dXx]`
- 金额：`[\\d,.]+万元?`

### 脱敏映射表

所有脱敏操作记录在 `kb_desensitization_mapping` 表中，保留原始值与脱敏值的映射关系，仅 admin_manage 权限可查看。
